# Referee 3 (LLM evaluation methodologist) — RSOS soundness review

**Overall recommendation**

Major revision.

The extension to 17 models is executed carefully, the estimand is stated far more honestly than in earlier rounds, and the repeated-elicitation machinery is reproducible. But the manuscript's framing of a "fixed elicitation protocol" is not accurate: the decoding-and-serialization harness varies systematically across models along four axes that are perfectly confounded with model identity (structured-output mechanism, completion budget, sampling temperature, and reasoning mode). For a soundness-based venue the paper does not yet establish that its cross-model differences — especially the within-Claude April→July "generation-to-generation belief shifts" it highlights — are properties of the models rather than of the harness. These issues are addressable with disclosure, a small number of targeted ablations, and a committed failure manifest, so I recommend major revision rather than reject. The provider code is accurately described in most specifics I checked, with a few concrete gaps noted below.

**Major comments**

1. **"Fixed elicitation protocol" overstates what is actually held fixed; the structured-output mechanism is confounded with model identity.** The abstract says results are obtained "under a fixed elicitation protocol" ([paper.qmd:10](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L10)), and the design is called "intentionally symmetric across providers" ([paper.qmd:137](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L137), [paper.qmd:141](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L141)). Only the *prompt* is fixed. The output-serialization harness is not: OpenAI uses strict JSON-schema Chat Completions ([providers.py:406-414](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L406-L414)); the July Claude models use native Anthropic strict-JSON-schema structured outputs ([providers.py:154-158](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L154-L158)); Claude through Opus 4.7 and Grok use LiteLLM *forced function calls* ([providers.py:231-237](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L231-L237)); and Gemini uses LiteLLM `json_object` mode, which enforces only that the output is a JSON object, **not** that it matches the belief schema ([providers.py:231-234](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L231-L234)). These are three constraint regimes of *different strength*, and constrained decoding is known to perturb both parseability and the numeric content of generations. Because the mechanism is fully collinear with provider (and, for Claude, with model generation), any cross-model difference in centers or widths is inseparable from a mechanism difference. This is sharpest for the within-provider comparison the paper explicitly advertises as "directly observable" ([paper.qmd:137](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L137)): claude-opus-4.7 (LiteLLM forced function call, 1200-token budget) versus claude-opus-4.8 (native strict schema, 32000-token budget, reasoning off) differ in the model *and* in at least three harness dimensions at once, so the "generation-to-generation belief shift" is not cleanly attributable to the model. The paper needs either (a) a same-model cross-mechanism ablation showing the harness does not move the pooled summaries, or (b) a redefinition of the estimand as "model-plus-provider-default-harness as deployed," with the confound stated as a first-order limitation rather than asserted away.

2. **Model version pinning: 16 of 17 models are floating aliases, which undermines reproducibility and confounds the April→July comparison.** Inspecting the alias maps, only `claude-haiku-4.5` resolves to a dated snapshot (`claude-haiku-4-5-20251001`, [providers.py:21](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L21)). Every other LiteLLM alias resolves to a floating name (`claude-opus-4-7`, `claude-sonnet-4-6`, `xai/grok-4.20-reasoning`, the three `gemini-*-preview` aliases, etc.; [providers.py:18-29](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L18-L29)), the three native-Claude aliases are floating ([providers.py:50-54](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L50-L54)), and the OpenAI models are passed as bare, un-dated names ([providers.py:376-405](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L376-L405)). Three of the April models are explicitly `-preview` aliases, which providers repoint frequently. This has two consequences the paper does not address. First, the elicited distributions are not reproducible against a fixed weight snapshot: a reader re-running the code months later may hit silently updated weights. Second, and worse for the central claim, the panel's headline "within-provider generation-to-generation belief shifts" ([paper.qmd:137](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L137)) could reflect the provider having repointed an April alias between April and July rather than a genuine model succession. The manuscript should disclose the exact resolved model IDs and snapshot dates for all 17 models, pin dated snapshots wherever the provider offers them, and record the resolution date/time so alias drift is at least bounded.

3. **The 58 re-elicited "infrastructure" failures: the replacement is defensible in principle, but the ignorability claim is too strong for the length-correlated subset, and the whole episode is not auditable from the committed artifacts.** The paper states that 58 runs failed and were re-elicited, and asserts "the replacement does not select on elicited values" because the failures are provider-side artifacts ([paper.qmd:160](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L160)). Two of the named failure modes — gpt-5.5 "exhausting the completion budget on reasoning" and three claude-sonnet-5 responses "exceeding a 32,000-token output cap" — are *length-correlated truncations*. A draw is dropped precisely when the model reasons longest, and reasoning length can correlate with the difficulty/uncertainty of the quantity and hence with the eventual number. So the surviving draws in those cells are a length-biased subsample, and re-drawing at the same budget re-randomizes but does not remove the bias; "does not select on elicited values" is therefore an assumption, not a fact, for this subset. Separately, I could not audit any of this from the repository: `rerun_failed_runs.py` overwrites the failed record in place ([rerun_failed_runs.py:146](/Users/maxghenis/llm-econ-beliefs/scripts/rerun_failed_runs.py#L146)), the committed `runs.jsonl` for all six July models now contains **0** error records, five of the six show exactly 390 request logs (no trace of the reruns), and there is no failures table in `paper/tables/`. The "uniformly 100%" success rate ([paper.qmd:158](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L158), and the `Success rate` column of every overview table) is thus a *post-replacement* rate; the first-attempt rate is unreported. The grok-4.3 "missing forced tool calls" failure is the very same failure mode that gave grok-4.20 a 12% v3 failure rate ([paper.qmd:160](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L160)), now absorbed silently. The paper should (i) commit a failure manifest (model, quantity, run index, failure type, attempts) and surface it as an appendix table, (ii) report first-attempt success by model, (iii) soften the ignorability claim to a conditional one for the truncation failures, and (iv) add an include-vs-exclude-reruns sensitivity for gpt-5.5 and claude-sonnet-5 showing the pooled center/width does not move.

4. **Reasoning-mode heterogeneity is broader than the paper discloses and is confounded with the headline uncertainty object.** The manuscript flags reasoning defaults only for the three July Claudes ([paper.qmd:146](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L146)). But the alias map shows the two Grok tiers are a *reasoning* variant (`xai/grok-4.20-reasoning`) versus an explicit *non-reasoning* variant (`xai/grok-4-1-fast-non-reasoning`), with grok-4.3 on a third default ([providers.py:22-24](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L22-L24)); the GPT-5.x models are reasoning models with their own defaults (gpt-5.5 "reasons far more than the 5.4 family", [providers.py:39-40](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L39-L40)); and Gemini carries its own "thinking" defaults. The paper's headline uncertainty object is the pooled *predictive* interval, whose width is driven by across-run sampling variation — exactly the quantity most sensitive to whether a model reasons before answering. Comparing pooled widths across models that differ in reasoning mode (e.g., grok-4.20 vs grok-4.1-fast, or opus-4.8-reasoning-off vs sonnet-5-adaptive) therefore compares a model attribute and a decoding-configuration attribute at once. At minimum the paper needs a per-model table of reasoning mode and a caveat that width comparisons are reasoning-mode-confounded; ideally reasoning mode enters the interpretation as a covariate.

5. **Decoding-settings reporting is incomplete for a methods paper, in ways that bear directly on the pooled interval.** (a) The native-Claude default *temperature* is never stated. The code correctly sends no sampling parameter for Fable 5 / Opus 4.8 / Sonnet 5 ([providers.py:154-158](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L154-L158)), and the paper says "variation comes from default sampling" ([paper.qmd:143](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L143)) — but if that provider default is not 1.0, these three models have systematically different draw-to-draw dispersion than the temperature-1.0 majority, which mechanically shifts their pooled widths. This is undisclosed and unbounded. (b) The gpt-5.5 completion budget of 8000 tokens ([providers.py:41-43](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L41-L43)) is omitted from the protocol section's budget list, which cites only 1200/4000/32000 ([paper.qmd:147](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L147)); the reader cannot reconstruct the 8000 value. (c) The OpenAI draws are batched (actual `batch_size = 5`, [run_v4_full_panel.py:97](/Users/maxghenis/llm-econ-beliefs/scripts/run_v4_full_panel.py#L97)) while re-elicited OpenAI draws use `n = 1` ([rerun_failed_runs.py:59](/Users/maxghenis/llm-econ-beliefs/scripts/rerun_failed_runs.py#L59)); the paper states only the cap "up to n = 8" ([paper.qmd:144](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L144)). (d) The Gemini `json_object` constraint is weaker than the schema/function-call constraints used elsewhere, and this asymmetry is not flagged. A single per-model harness table (resolved ID, snapshot date, mechanism, temperature, reasoning mode, budget, draws-per-request) would close all of (a)-(d) at once and is standard for this literature.

6. **Contamination framing: memory-only elicitation makes the benchmark comparison partly circular, and compounds the temporal confound.** The main prompt is explicitly memory-only — answer from background knowledge, no tools, no literature review ([paper.qmd:49](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L49)) — which is the right design choice for a well-defined estimand, and the near-zero tool uptake in the archived arm supports it. But it means the elicited numbers are, mechanically, recall of parameter values the models saw in training. The benchmark check "most labor-and-tax centers lie inside rough review ranges" ([paper.qmd:211-217](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L211-L217)) is then not independent corroboration: the models were trained on the same CBO/Saez-Slemrod-Giertz literature that defines those ranges, so agreement is the *expected* outcome under faithful memorization, not evidence the elicitation recovers a correct value. The paper should add this circularity caveat and reframe Table 3 as a consistency-with-recalled-literature check. This also compounds Majors 1-2: the July models have later training cutoffs, different corpora, and different RLHF shaping, so the April→July shifts already confounded by mechanism (Major 1) and by alias drift (Major 2) are *additionally* confounded by training-data vintage. Finally, the requested `citations` field ([paper.qmd:58](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L58)) is a cheap, currently-unused contamination probe: reporting the fraction of elicited citations that correspond to real works would directly characterize recall fidelity versus confabulation.

**Minor comments**

- The 100% parse claim ([paper.qmd:158](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L158)) should be labeled post-re-elicitation, with the first-attempt rate reported alongside.
- The gemini-3.5-flash near-zero centers on both sign-sensitive quantities ([paper.qmd:215](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L215)) are interpreted purely as residual sign instability; given that Gemini alone runs under the weaker `json_object` constraint, a mechanism/constraint interaction cannot be excluded and should be acknowledged.
- The budget is described as "a truncation guard, not part of the elicitation content" ([paper.qmd:147](/Users/maxghenis/llm-econ-beliefs/paper/paper.qmd#L147)). That is true only when it does not bind; when it binds it *is* a content-selection mechanism (it produced the gpt-5.5 and sonnet-5 failures), so the phrasing is slightly too reassuring.
- The providers module docstring reads "adapters for running prompts against locally available CLIs" ([providers.py:1](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L1)), but the panel paths are HTTP/SDK, not CLIs; a reproducibility-minded reader will trip on this stale description.
- Consider stating whether the OpenAI system prompt ("Follow the user's instructions exactly and return only the final answer.", [providers.py:397-400](/Users/maxghenis/llm-econ-beliefs/llm_econ_beliefs/providers.py#L397-L400)) has any counterpart on the LiteLLM and Anthropic paths; an OpenAI-only system message is another small asymmetry across the "symmetric" harness.

**Concrete revision requests**

1. Add a single per-model harness table with one row per model: resolved API model ID, snapshot date or "floating alias", structured-output mechanism (strict JSON schema / forced function call / json_object), sampling temperature (or "provider default = X"), reasoning mode, completion budget, and draws-per-request.
2. Run a cross-mechanism ablation on at least one Claude model available on both the LiteLLM forced-function-call path and the native-Anthropic strict-schema path, and report pooled-center and pooled-width deltas; analogously, re-run one gpt-5.5 cell at 1200 vs 8000 tokens to bound the budget effect. Use these to defend (or retire) the "generation-to-generation belief shift" reading.
3. Commit a failure manifest (model, quantity, run index, failure type, number of attempts) preserving the discarded records, add it as an appendix table, report first-attempt success rates by model, and add an include-vs-exclude-reruns sensitivity for gpt-5.5 and claude-sonnet-5.
4. Disclose exact resolved model IDs and resolution dates; pin dated snapshots where available; add alias drift as an explicit reproducibility limitation and as a confound for the April→July comparison.
5. Soften "does not select on elicited values" to a claim conditional on truncation being independent of the elicited value, and support it with the sensitivity check in request 3.
6. Add the training-data-recall/circularity caveat to the benchmark section, disclose the native-Claude default temperature, and (optionally) report a citation-validity rate as a contamination probe.
